# AIWMR Training App — PWA

**Progressive Web App for AIWMR Online Certification Courses**

Stack: React 18 · TypeScript · Vite + SWC · vite-plugin-pwa · Supabase · Razorpay

---

## Project Overview

**What this is:**
A production-ready Progressive Web App (PWA) for Ashrita Institute for Waste Management & Research (AIWMR). Users can browse courses, register with Razorpay payments, attend online modules, mark QR-based attendance, and download certificates.

**Key features:**
- ✅ Full PWA (installable, works offline, no browser chrome when installed)
- ✅ Auto-appearing Install banner (custom UI, controlled via `usePWAInstall` hook)
- ✅ Real Supabase + Razorpay integration points (currently in demo mode with clear TODOs)
- ✅ Full TypeScript, proper folder structure, zero unused imports
- ✅ Full-viewport layout (no phone wrapper — this is the real app)
- ✅ Safe area insets for notched phones
- ✅ Service worker auto-generated by vite-plugin-pwa

## Quick Start

```bash
npm install
cp .env.example .env
# Edit .env with your Supabase URL + anon key, Razorpay key
npm run dev          # opens at http://localhost:3000
npm run build        # production build to /dist
```

**Demo mode:** Works immediately without any .env — shows demo data, accepts any login credentials, simulates payments. Add real credentials to switch to production mode.

## How to Connect Real Data

### Step 1: Supabase Database

Copy this SQL into your Supabase SQL editor and run it:

```sql
-- ── User Profiles ──────────────────────────────────────────────────────────

create table profiles (
  id           uuid references auth.users primary key,
  name         text not null,
  phone        text,
  organization text,
  designation  text,
  role         text default 'trainee' 
               check (role in ('trainee','corporate','government','trainer','admin')),
  created_at   timestamptz default now()
);

-- Auto-create profile on signup
create or replace function handle_new_user()
returns trigger as $$
begin
  insert into profiles (id, name, email, phone) 
  values (
    new.id, 
    new.raw_user_meta_data->>'name',
    new.email,
    new.raw_user_meta_data->>'phone'
  );
  return new;
end;
$$ language plpgsql security definer;

create trigger on_auth_user_created
  after insert on auth.users
  for each row execute procedure handle_new_user();

-- Add email to profiles
alter table profiles add column email text;

-- ── Courses ─────────────────────────────────────────────────────────────────

create table courses (
  id         serial primary key,
  title      text not null,
  subtitle   text,
  duration   text,
  fee        integer not null,
  seats      integer default 15,
  filled     integer default 0 check (filled <= seats),
  mode       text check (mode in ('Online','Offline','Hybrid')),
  start_date date,
  badge      text,
  trainer    text,
  category   text,
  color      text,
  icon       text,
  topics     text[],
  created_at timestamptz default now()
);

-- ── Batches ─────────────────────────────────────────────────────────────────

create table batches (
  id        serial primary key,
  course_id integer references courses(id) on delete cascade,
  label     text not null,
  date      date,
  time_slot text,
  seats     integer,
  created_at timestamptz default now()
);

-- ── Registrations ───────────────────────────────────────────────────────────

create table registrations (
  id              uuid default gen_random_uuid() primary key,
  user_id         uuid references profiles(id) on delete cascade,
  course_id       integer references courses(id) on delete cascade,
  batch_id        integer references batches(id),
  payment_status  text default 'pending' 
                  check (payment_status in ('pending','paid','failed','refunded')),
  payment_id      text,
  razorpay_order_id text,
  razorpay_signature text,
  registration_id text unique not null,
  qr_code         text,
  created_at      timestamptz default now()
);

-- Auto-generate registration ID
create or replace function generate_registration_id()
returns trigger as $$
begin
  new.registration_id := 'AIWMR-' || 
    to_char(now(), 'YYYY') || '-' || 
    lpad(nextval('registrations_id_seq')::text, 4, '0');
  return new;
end;
$$ language plpgsql;

create trigger set_registration_id
  before insert on registrations
  for each row execute procedure generate_registration_id();

-- ── Modules ─────────────────────────────────────────────────────────────────

create table modules (
  id          serial primary key,
  course_id   integer references courses(id) on delete cascade,
  title       text not null,
  type        text check (type in ('video','pdf','quiz','assignment')),
  duration    text,
  order_index integer,
  video_url   text,  -- Supabase Storage URL
  pdf_url     text,  -- Supabase Storage URL
  description text,
  created_at  timestamptz default now()
);

-- ── User Progress ───────────────────────────────────────────────────────────

create table user_progress (
  id          uuid default gen_random_uuid() primary key,
  user_id     uuid references profiles(id) on delete cascade,
  module_id   integer references modules(id) on delete cascade,
  status      text default 'locked' 
              check (status in ('locked','in-progress','completed')),
  completed_at timestamptz,
  updated_at  timestamptz default now(),
  unique(user_id, module_id)
);

-- ── Attendance ──────────────────────────────────────────────────────────────

create table attendance (
  id              uuid default gen_random_uuid() primary key,
  registration_id uuid references registrations(id) on delete cascade,
  session_date    date not null,
  marked_at       timestamptz default now(),
  unique(registration_id, session_date)
);

-- ── Session QR Codes ────────────────────────────────────────────────────────

create table session_qr_codes (
  id         uuid default gen_random_uuid() primary key,
  course_id  integer references courses(id) on delete cascade,
  batch_id   integer references batches(id),
  session_date date not null,
  qr_code    text unique not null,
  expires_at timestamptz,
  created_at timestamptz default now()
);

-- ── Certificates ────────────────────────────────────────────────────────────

create table certificates (
  id              uuid default gen_random_uuid() primary key,
  registration_id uuid references registrations(id) on delete cascade unique,
  cert_id         text unique not null,
  issued_at       timestamptz default now(),
  pdf_url         text
);

-- Auto-generate cert ID
create or replace function generate_cert_id()
returns trigger as $$
begin
  new.cert_id := 'AIWMR-CERT-' || 
    to_char(now(), 'YYYY') || '-' || 
    lpad(floor(random() * 10000)::text, 4, '0');
  return new;
end;
$$ language plpgsql;

create trigger set_cert_id
  before insert on certificates
  for each row execute procedure generate_cert_id();

-- ── Row Level Security ──────────────────────────────────────────────────────

alter table profiles      enable row level security;
alter table registrations enable row level security;
alter table user_progress enable row level security;
alter table attendance    enable row level security;
alter table certificates  enable row level security;

-- Profiles: users can read own, admins read all
create policy "Users read own profile" on profiles
  for select using (auth.uid() = id);

create policy "Admins read all profiles" on profiles
  for select using (
    exists (select 1 from profiles where id = auth.uid() and role = 'admin')
  );

-- Registrations: users read own
create policy "Users read own registrations" on registrations
  for select using (auth.uid() = user_id);

create policy "Users insert own registrations" on registrations
  for insert with check (auth.uid() = user_id);

-- User progress: users read/update own
create policy "Users read own progress" on user_progress
  for select using (auth.uid() = user_id);

create policy "Users update own progress" on user_progress
  for all using (auth.uid() = user_id);

-- Attendance: users read own
create policy "Users read own attendance" on attendance
  for select using (
    registration_id in (
      select id from registrations where user_id = auth.uid()
    )
  );

create policy "Users insert own attendance" on attendance
  for insert with check (
    registration_id in (
      select id from registrations where user_id = auth.uid()
    )
  );

-- Certificates: users read own
create policy "Users read own certificates" on certificates
  for select using (
    registration_id in (
      select id from registrations where user_id = auth.uid()
    )
  );

-- Public read for courses, modules, batches (no auth required)
alter table courses enable row level security;
alter table modules enable row level security;
alter table batches enable row level security;

create policy "Public read courses" on courses for select using (true);
create policy "Public read modules" on modules for select using (true);
create policy "Public read batches" on batches for select using (true);
```

```typescript
import { Document, Page, Text, View, StyleSheet, pdf } from '@react-pdf/renderer';

const styles = StyleSheet.create({
  page: { padding: 60, backgroundColor: '#f7f3ec' },
  title: { fontSize: 32, fontFamily: 'Times-Bold', color: '#1a3a2a', textAlign: 'center' },

const CertificatePDF = ({ name, course, date, certId }) => (
  <Document>
    <Page size="A4" orientation="landscape" style={styles.page}>
      <Text style={styles.title}>Certificate of Completion</Text>
      <Text style={styles.name}>{name}</Text>
      <Text style={styles.course}>{course}</Text>
      <Text style={styles.date}>Issued: {date}</Text>
      <Text style={styles.certId}>ID: {certId}</Text>
    </Page>
  </Document>
);

export const generateCertificate = async (data) => {
  const blob = await pdf(<CertificatePDF {...data} />).toBlob();
  const url = URL.createObjectURL(blob);
  
  // Upload to Supabase Storage
  const fileName = `${data.certId}.pdf`;
  const { error } = await supabase.storage
    .from('certificates')
    .upload(fileName, blob);
  
  if (!error) {
    const { data: { publicUrl } } = supabase.storage
      .from('certificates')
      .getPublicUrl(fileName);
    
    // Save URL to certificates table
    await supabase.from('certificates').update({ pdf_url: publicUrl }).eq('cert_id', data.certId);
  }
  
  return url; // for immediate download
};
```

## PWA Features Explained

### Install Banner

The `InstallBanner` component listens for the `beforeinstallprompt` event via `usePWAInstall.ts`. When the browser decides the app is installable (usually after 30 seconds of use), the banner appears automatically at the top.

The user taps "Install" → `promptInstall()` triggers the native browser install dialog → app gets added to home screen.

**To test:**
1. `npm run build && npm run preview`
2. Open in Chrome on Android or Desktop
3. Wait 30 seconds
4. Banner appears

**Customization:**
Edit `src/components/InstallBanner.tsx` to change the design. The logic in `usePWAInstall.ts` handles all the browser API calls.

### Service Worker

`vite-plugin-pwa` auto-generates `/dist/sw.js` on every build. It caches:
- All static assets (JS, CSS, fonts, images)
- Supabase API responses (NetworkFirst strategy)
- Google Fonts (CacheFirst strategy)

**To verify:**
1. Build: `npm run build`
2. Check `/dist/sw.js` exists
3. Open DevTools → Application → Service Workers

### Manifest

Generated in `vite.config.ts`. Controls:
- App name, short name, description
- Theme color (forest green `#1a3a2a`)
- Display mode (`standalone` = no browser chrome)
- Icons (72px to 512px)
- Shortcuts (My Courses, Attendance)

**To customize:**
Edit the `manifest` object in `vite.config.ts`.

### Offline Mode

Once installed and cached:
- App loads instantly even with no internet
- Static pages work fully offline
- Supabase calls fail gracefully (show cached data or error message)

**To add offline fallback UI:**
In `App.tsx`, detect network status:
```typescript
const [online, setOnline] = useState(navigator.onLine);
useEffect(() => {
  const handleOnline  = () => setOnline(true);
  const handleOffline = () => setOnline(false);
  window.addEventListener('online', handleOnline);
  window.addEventListener('offline', handleOffline);
  return () => {
    window.removeEventListener('online', handleOnline);
    window.removeEventListener('offline', handleOffline);
  };
}, []);

{!online && <div className="offline-banner">You're offline. Some features may not work.</div>}
```

## Security Checklist

- ✅ Never commit `.env` to git (`.gitignore` includes it)
- ✅ Use anon key (not service role key) in frontend
- ✅ Row Level Security (RLS) enabled on all Supabase tables
- ✅ Razorpay Key Secret never exposed in frontend (used only in Edge Function)
- ✅ Payment verification happens server-side (via Supabase Edge Function)
- ✅ User can only access their own data (RLS policies enforce `auth.uid() = user_id`)

**Don't:**
- ❌ Put Supabase service role key in `.env` (it has admin access)
- ❌ Put Razorpay Key Secret in `.env` (use Edge Function instead)
- ❌ Trust payment data from frontend (verify signature server-side)
- ❌ Skip RLS policies (users could access other users' data)

**Build fails with TypeScript errors:**

```bash
npm run typecheck
```

**PWA install prompt doesn't appear:**
- Only works on HTTPS (or localhost)

- Browser decides when app is "installable"

**Supabase queries return empty:**

- Check browser console for errors

```bash
# DevTools → Application → Service Workers → Unregister

# Or:
npm run build && npm run preview
```

**Icons don't show:**

- Run `npm run build` to regenerate manifest
